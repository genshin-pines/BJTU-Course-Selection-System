# 登录状态管理技术方案

> 技术栈：Spring Boot + Vue 3 + MySQL + Redis  
> 适用场景：需要持久化登录状态、单设备登录、管理员监控在线用户

---

## 一、登录状态持久化（刷新页面不退出）

### 1.1 问题分析

Token 存储位置决定了刷新后能否保持登录：

| 存储位置 | 刷新保留 | XSS 安全性 | CSRF 安全性 |
|----------|:--------:|:----------:|:----------:|
| `localStorage` | ✅ | ❌ 易被脚本读取 | ✅ |
| `sessionStorage` | ❌ | ❌ | ✅ |
| JS 可读 Cookie | ✅ | ❌ | ❌ |
| **HttpOnly Cookie** | ✅ | ✅ | ❌ 需额外处理 |

### 1.2 推荐方案：双 Token 机制

```
Access Token   — 有效期 15~30 分钟  →  存在前端内存（Vuex/Pinia）
Refresh Token  — 有效期 7~30 天     →  存在 HttpOnly Cookie
```

**为什么不用 localStorage？**
- XSS 攻击可以直接读取 localStorage 中的 Token
- HttpOnly Cookie 对 JavaScript 完全不可见，即使被注入脚本也无法窃取

### 1.3 完整流程

```
┌──────────┐          ┌──────────────┐          ┌──────────┐
│  浏览器    │          │  Vue 前端     │          │  Spring   │
│           │          │              │          │  Boot     │
└─────┬─────┘          └──────┬───────┘          └─────┬─────┘
      │                       │                        │
      │  ① 用户刷新页面         │                        │
      │ ──────────────────────>│                        │
      │                       │                        │
      │                       │ ② Pinia 中 Access Token 为空
      │                       │                        │
      │                       │ ③ Cookie 自动携带        │
      │                       │ Refresh Token           │
      │                       │ ──POST /api/auth/refresh──>
      │                       │                        │
      │                       │                        │ ④ 校验 Refresh Token
      │                       │                        │ ⑤ 生成新 Access Token
      │                       │ <──── 新 Access Token ────
      │                       │                        │
      │                       │ ⑥ 存回 Pinia 状态        │
      │                       │ ⑦ 重试之前失败的请求      │
      │                       │                        │
```

用户完全无感知，刷新后自动恢复登录态。

### 1.4 后端实现要点

#### Refresh Token 存入 Cookie

```java
// 登录成功时设置 Cookie
ResponseCookie refreshCookie = ResponseCookie.from("refresh_token", refreshToken)
    .httpOnly(true)           // JS 不可读
    .secure(true)             // 仅 HTTPS 传输
    .sameSite("Strict")       // 防 CSRF
    .path("/api/auth")        // 仅认证接口携带
    .maxAge(7 * 24 * 60 * 60) // 7 天过期
    .build();
response.addHeader(HttpHeaders.SET_COOKIE, refreshCookie.toString());
```

#### Refresh 接口

```java
@PostMapping("/auth/refresh")
public Result<LoginVO> refresh(
    @CookieValue("refresh_token") String refreshToken
) {
    // 1. 校验 Refresh Token 有效性
    // 2. 查 Redis 确认会话未被踢
    // 3. 查 DB 确认 token_version 匹配（防止多设备登录）
    // 4. 生成新的 Access Token
    // 5. 可选：滑动刷新 Refresh Token（重置过期时间）
    return Result.ok(new LoginVO(accessToken, userInfo));
}
```

### 1.5 前端实现要点

#### Axios 拦截器（自动刷新）

```javascript
// 响应拦截器
axios.interceptors.response.use(
  response => response,
  async error => {
    if (error.response?.status === 401 && !error.config._retry) {
      error.config._retry = true
      try {
        // 调 refresh 接口获取新 Access Token
        const { data } = await axios.post('/api/auth/refresh')
        // 更新存储
        authStore.setToken(data.accessToken)
        // 重试原请求
        error.config.headers.Authorization = `Bearer ${data.accessToken}`
        return axios(error.config)
      } catch {
        // refresh 也失败了，跳登录页
        router.push('/login')
      }
    }
    return Promise.reject(error)
  }
)
```

#### 路由守卫

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  if (to.meta.requiresAuth && !authStore.token) {
    // 尝试 refresh
    authStore.tryRefresh().then(() => {
      authStore.token ? next() : next('/login')
    })
  } else {
    next()
  }
})
```

## 二、同一账号只能在一处登录

### 2.1 方案对比

| 方案 | 实时性 | 复杂度 | 适用场景 |
|------|:------:|:------:|----------|
| Token 版本号（DB 字段） | 非实时 | 低 | 小项目 |
| **Redis 会话管理** | 实时 | 中 | **推荐** |
| WebSocket 主动推送 | 实时 | 高 | 高安全场景 |

### 2.2 推荐方案：Redis + Token 版本号（双重保障）

#### 数据结构

**Redis Key：** `session:{userId}`

```json
{
    "tokenId": "uuid-v4",
    "username": "2022111111",
    "loginIp": "10.2.3.4",
    "userAgent": "Chrome/125.0 Windows",
    "loginTime": "2026-06-10 21:00:00",
    "lastActiveTime": "2026-06-10 21:17:30",
    "accessToken": "eyJhbG...",
    "refreshToken": "eyJhbG..."
}
TTL: 7 天
```

**MySQL User 表额外字段：**

```sql
ALTER TABLE user ADD COLUMN token_version INT DEFAULT 0;
```

#### 登录流程

```
  用户A 在 Chrome 登录
      ↓
  ① 生成新 tokenId（UUID）
  ② Redis SET session:{userId} → {tokenId, ...}
  ③ DB UPDATE user SET token_version = token_version + 1
  ④ JWT payload 中写入 tokenId + tokenVersion
  ⑤ 返回双 Token

  用户A 在 Safari 再登录
      ↓
  ① 生成新 tokenId
  ② Redis SET session:{userId} → {新 tokenId, ...}（覆盖旧值）
  ③ DB token_version 再次 +1
  ④ Chrome 的 Token 已失效（version 不匹配）
```

#### 请求校验过滤器

```java
// JWT 校验过滤器核心逻辑
public void doFilter(request, response, chain) {
    // 1. 解析 JWT 获取 userId, tokenId, tokenVersion
    String userId = jwtUtil.getUserId(token);
    String tokenId = jwtUtil.getTokenId(token);
    int tokenVersion = jwtUtil.getTokenVersion(token);

    // 2. 查 Redis
    Session session = redisTemplate.opsForValue()
        .get("session:" + userId);

    // 3. 双重校验
    if (session == null) {
        throw new TokenExpiredException("会话已过期，请重新登录");
    }
    if (!session.getTokenId().equals(tokenId)) {
        throw new KickOutException("该账号已在其他设备登录");
    }
    if (session.getTokenVersion() != tokenVersion) {
        throw new KickOutException("登录凭证已失效");
    }

    // 4. 更新最后活跃时间（异步）
    asyncUpdateLastActive(userId);

    // 5. 放行
    chain.doFilter(request, response);
}
```

## 三、管理端监控用户状态

### 3.1 功能需求

- 查看所有在线用户列表
- 显示登录时间、IP、设备、活跃状态
- 支持强制踢出指定用户
- 在线人数统计

### 3.2 后端接口设计

#### 获取在线用户列表

```
GET /admin/sessions
Response:
{
    "total": 15,
    "online": 3,
    "list": [
        {
            "userId": 1,
            "username": "2022111111",
            "loginIp": "10.2.3.4",
            "userAgent": "Chrome / Windows",
            "loginTime": "2026-06-10 15:30:00",
            "lastActiveTime": "2026-06-10 21:17:30",
            "status": "online"    // online | idle | offline
        }
    ]
}
```

#### 踢出用户

```
POST /admin/sessions/kick
Body: { "userId": 1 }

内部逻辑：
  ① Redis DEL session:{userId}
  ② DB UPDATE user SET token_version = token_version + 1
  ③ 该用户下次请求时 JWT 校验失败 → 强制跳登录
```

#### 统计接口

```
GET /admin/sessions/stats
Response:
{
    "onlineCount": 3,
    "todayActive": 42,
    "peakToday": 8,
    "peakTime": "2026-06-10 14:30"
}
```

### 3.3 后端实现：扫描 Redis

```java
@Service
public class SessionMonitorService {

    public List<SessionVO> getAllSessions() {
        List<SessionVO> result = new ArrayList<>();

        // 扫描所有 session:* key
        Set<String> keys = redisTemplate.keys("session:*");
        if (keys == null) return result;

        for (String key : keys) {
            Session session = redisTemplate.opsForValue().get(key);
            if (session == null) continue;

            SessionVO vo = new SessionVO();
            vo.setUserId(session.getUserId());
            vo.setUsername(session.getUsername());
            vo.setLoginIp(session.getLoginIp());
            vo.setUserAgent(session.getUserAgent());
            vo.setLoginTime(session.getLoginTime());
            vo.setLastActiveTime(session.getLastActiveTime());
            vo.setStatus(determineStatus(session.getLastActiveTime()));
            result.add(vo);
        }
        return result;
    }

    private String determineStatus(LocalDateTime lastActive) {
        long minutes = ChronoUnit.MINUTES.between(lastActive, LocalDateTime.now());
        if (minutes < 5) return "online";
        if (minutes < 30) return "idle";
        return "offline";
    }
}
```

## 四、增强：心跳与实时推送（可选）

### 4.1 心跳机制

前端每 5 分钟自动发送一次心跳请求，更新 `lastActiveTime`：

```javascript
// 在 Pinia Store 中
let heartbeatTimer

function startHeartbeat() {
  heartbeatTimer = setInterval(() => {
    axios.post('/api/auth/heartbeat')
  }, 5 * 60 * 1000)
}

function stopHeartbeat() {
  clearInterval(heartbeatTimer)
}
```

### 4.2 WebSocket 实时通知

当用户被踢下线时，通过 WebSocket 主动推送通知前端：

```
  管理员点 [踢下线]
       ↓
  后端删除 Redis 会话 + 递增 token_version
       ↓
  通过 WebSocket 推送消息给该用户
  { "type": "kick_out", "message": "账号已在其他设备登录" }
       ↓
  前端收到 → 清除状态 → 跳转登录页 → 提示用户
```

## 六、要点速查表

| 需求 | 方案 | 关键技术点 |
|------|------|-----------|
| 刷新不退出 | 双 Token | Access(内存) + Refresh(HttpOnly Cookie) |
| 自动续期 | 响应拦截器 | Axios 拦截 401 → 调 refresh → 重试 |
| 单设备登录 | Redis 会话 | session:{userId} 存唯一 tokenId |
| 双重保障 | token_version | DB 字段配合，即使 Redis 故障也能兜底 |
| 在线监控 | 扫描 Redis | 遍历 session:* key，展示在线列表 |
| 强制踢人 | 删 Key + 递增版本 | 下次请求直接 401 |
| 心跳机制 | 定时 POST | 每 5 分钟更新 lastActiveTime |
| 实时推送 | WebSocket | 管理员踢人实时通知前端 |
