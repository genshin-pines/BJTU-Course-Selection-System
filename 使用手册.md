## 系统概述

BJTU 课程评价系统是一款面向北京交通大学学生的课程匿名评价平台。

- **学生**：搜索课程、浏览评价、发布/修改自己的评价、点赞点踩、举报不当评价
- **管理员**：审核评价、管理课程/教师/开课实例数据、管理管理员账号、处理举报

系统支持敏感词自动过滤，评价发布后立即展示（先发后审），管理员可进行后续审核。

## 学生端

### 注册账号

1. 打开系统首页，点击右上角 **登录**
2. 切换到 **注册** 标签页
3. 填写信息：
   - **邮箱**：格式为 `8位学号@bjtu.edu.cn`（如 `20221111@bjtu.edu.cn`）
   - **学号**：与邮箱前缀一致（如 `20221111`）
   - **密码**：至少 6 位
   - **确认密码**：再次输入密码
4. 点击 **发送验证码**，查收邮箱获取 6 位验证码
5. 输入验证码，点击 **注册**
6. 注册成功后自动跳转到登录页

> 注意：邮箱需为 BJTU 校园邮箱格式，系统会校验邮箱前缀与学号一致。

### 登录

1. 点击右上角 **登录**
2. 输入 **学号** 和 **密码**
3. 点击 **登录**
4. 登录成功后自动跳转首页

### 搜索课程

在首页可以使用多种方式搜索课程：

| 筛选项 | 说明 |
|--------|------|
| 搜索框 | 输入课程名称或代码进行模糊搜索 |
| 学院 | 下拉选择学院 |
| 教师 | 下拉选择教师 |
| 最低评分 | 只显示综合评分高于此值的课程 |
| 标签 | 选择标签筛选（如"给分好"、"水课"等） |

点击 **搜索** 或直接回车即可查看结果。课程列表按评价数量排序。

### 查看课程详情与评价

1. 在课程列表中点击任一课程卡片
2. 进入课程详情页，可以看到：
   - 课程基本信息（名称、代码、教师、学分）
   - 综合评分和分项评分（给分、授课、作业）
   - 所有评价列表
3. 可以对评价进行排序：
   - **质量优先**（默认）：综合点赞数、内容长度、评价完整度
   - **最新优先**：按发布时间倒序
   - **高分优先**：按综合评分降序
   - **有用优先**：按点赞数降序
   - **争议优先**：按投票总数降序

### 发布评价

1. 在课程详情页点击 **写评价** 按钮
2. 对各维度打分（1-5 分）：
   - **给分情况**：课程给分高低
   - **授课质量**：教师授课水平
   - **作业轻松度**：课业负担
3. 填写评价内容（必填）
4. 选填：
   - 考核方式（如"闭卷考试+平时作业"）
   - 重点章节
   - 是否可带资料（Cheat Sheet）
   - 复习建议
   - 标签（如"给分好"、"干货多"等）
5. 点击 **提交评价**
6. 提交后：
   - 无敏感词 → 立即发布，所有人可见
   - 含敏感词 → 自动隐藏，需修改后重新提交

> 同一学生对同一课程只能发布一条评价。

### 修改评价

1. 在已评价的课程详情页，点击 **修改评价**
2. 修改内容和评分
3. 点击 **保存修改**
4. 修改后的评价立即生效，进入审核流程

### 删除评价

在评价管理页可删除自己的评价。

### 点赞/点踩

- 在评价下方点击 **有用** 点赞，再次点击取消
- 点击 **没用** 点踩，再次点击取消
- 不能给自己的评价投票

### 举报评价

1. 对任何评价点击 **举报**
2. 填写举报原因
3. 提交举报，管理员将审核处理

## 管理员端

### 管理员登录

访问 `/admin/login` 或首页底部管理员入口。

**默认账号：**

| 账号 | 密码 | 角色 | 权限范围 |
|------|------|------|----------|
| admin | 123456 | SUPER_ADMIN | 全部权限 |
| dept_op | 123456 | DEPT_OP | 院系管理（可配学院范围） |
| auditor | 123456 | AUDITOR | 审核员 |

### 评价审核

登录后默认进入 **评价审核** 页面，显示所有待审核评价。

- 待审核列表展示：课程、教师、评分、内容、用户
- **通过**：评价变为"审核通过"状态，正常展示
- **拒绝**：需填写拒绝原因，评价变为"已隐藏"状态

> "先发后审"：学生提交的评价状态为"已发布待审核"，发布后立即可见，同时出现在待审核列表中。

### 全部评价

**全部评价** 标签页可以查看和管理所有评价：

- **筛选**：按状态（已发布待审核 / 审核通过 / 已隐藏）、课程名称、教师姓名、学院、时间范围
- **排序**：按发布时间倒序
- **删除**：点击删除按钮可永久删除评价（不可恢复）
- **分页**：底部可切换每页条数和翻页

### 举报管理

查看和处理用户举报：

- 列表显示：被举报评价内容、举报原因、举报用户、状态
- **采纳**：填写处理原因，隐藏/删除对应评价
- **驳回**：填写驳回原因，举报标记为已处理

### 标签管理

- 输入新标签名，点击 **添加标签**
- 点击标签上的 × 可删除标签
- 标签用于学生评价时的勾选，便于分类浏览

### 基础数据管理

在 **基础数据** 标签页，有三个子标签：

#### 课程管理
- 查看、新增、编辑、删除课程
- 必填：课程代码、课程名称、学分
- 支持 Excel 批量导入

#### 教师管理
- 查看、新增、编辑、删除教师
- 必填：教师姓名

#### 开课实例管理
- 每个"课程+教师+学期+班级"的组合为一个开课实例
- 学生评价绑定到具体开课实例
- 支持新增、编辑、删除

### 管理员账号管理

仅有 SUPER_ADMIN 可以访问此功能。

- **创建管理员**：输入用户名和密码
- **修改角色**：SUPER_ADMIN / DEPT_OP / AUDITOR
- **设置院系范围**（DEPT_OP）：限制该管理员只能查看/管理指定学院的数据
- **重置密码**：为管理员设置新密码
- **删除管理员**：移除管理员账号

## 部署说明

### 环境要求
- JDK 17+
- Maven 3.6+
- Node.js 18+
- MySQL 8.0+

### 数据库初始化
```sql
CREATE DATABASE IF NOT EXISTS bjtu_review DEFAULT CHARACTER SET utf8mb4;
SOURCE backend/src/main/resources/db/schema.sql;
```

### 启动后端（端口 8889）
```bash
cd backend
mvn spring-boot:run
```

### 启动前端（端口 8081）
```bash
cd frontend
npm install
npm run dev
```

### 配置说明

修改 `backend/src/main/resources/application.yml`：
- `spring.datasource.password` → 你的 MySQL 密码
- `spring.mail.username` → 发送验证码的 QQ 邮箱
- `spring.mail.password` → QQ 邮箱授权码

浏览器访问 `http://localhost:8081` 即可使用。
