# 北京交通大学课程评价系统使用手册

本文档面向学生用户、后台管理员和部署维护人员，说明系统的主要功能与常用操作。

## 访问入口

本地开发环境：

```text
http://localhost:8081
```

远端部署环境：

```text
http://101.32.222.213
```

后台入口：

```text
/admin/login
```

## 学生端使用

### 注册与登录

1. 打开首页，进入登录/注册页面。
2. 注册时填写学号、姓名、邮箱和密码。
3. 邮箱验证码由系统 SMTP 邮箱发送，验证码有效期为 5 分钟。
4. 登录成功后，系统会保存当前会话；刷新页面不会退出登录。
5. 同一账号仅允许一处登录，新登录会使旧会话失效。

| 学号 | 密码 |
| --- | --- |

| 2022111112 | 123456 |

### 搜索课程

首页支持以下筛选方式：

- 关键词：课程代码、课程名称、教师姓名。
- 学院：按课程所属学院筛选。
- 教师：按授课教师筛选。
- 评分：按综合评分、给分、授课质量、作业轻松度筛选。
- 标签：按评价标签筛选，可用于快速找到“给分好”“作业少”等课程。
- 排序：可按评价数量、评分维度、课程代码等排序。

### 查看课程详情

点击课程卡片进入课程详情页，可以查看：

- 课程基础信息。
- 教师信息。
- 综合评分、给分、授课质量、作业轻松度。
- 已发布评价列表。
- 评价标签、学习建议、考核方式和重点章节。

### 发布评价

1. 登录学生账号。
2. 在课程详情页点击“写评价”。
3. 填写评分、评价内容、学习建议、考核方式和标签。
4. 提交后系统会使用匿名身份发布评价。
5. 如果内容触发敏感词规则，评价会被隐藏或进入治理流程。

注意：

- 每名学生对同一个开课实例只能保留一条有效评价。
- 学生只能编辑或删除自己的评价。
- 管理员账号不能在学生端发布评价、点赞或举报。

### 投票与举报

评价卡片支持：

- 点赞：表示评价有帮助。
- 点“没用”：表示评价参考价值低。
- 举报：认为评价违规时提交举报原因。

系统会阻止给自己的评价投票或举报自己的评价。

### 个人中心

个人中心可查看自己发布过的评价，并进行编辑或删除。

### 评价审核

审核员和超级管理员可以：

- 查看待审核评价。
- 通过评价。
- 拒绝或隐藏评价，并填写操作原因。
- 查看评价内容、评分、课程、教师和匿名用户信息。

操作会写入审计日志。

### 举报处理

审核员和超级管理员可以：

- 查看待处理举报。
- 采纳举报：删除或隐藏对应评价。
- 驳回举报：保留评价。
- 填写处理原因。

处理结果会写入审计日志。

### 数据维护

超级管理员和院系维护员可以：

- 新增、编辑、删除课程基础信息。
- 新增、编辑、删除教师。
- 新增、编辑、删除开课实例。
- 维护评价标签。
- 通过 CSV 批量导入课程和教师数据。

导入模板字段：

```csv
courseCode,courseName,credit,department,teacherName,teacherDepartment
```

### 管理员账号

仅超级管理员可以：

- 新增管理员账号。
- 修改管理员角色和所属院系。
- 重置管理员密码。
- 删除非当前登录管理员账号。

## 运维操作


```powershell
.\start.ps1 -CheckOnly
```


### 查看远端后端日志

```bash
sudo journalctl -u bjtu-review -f
```

### 重启远端后端

```bash

```

```bash

```

常用环境变量：

```env
SPRING_DATASOURCE_URL=jdbc:mysql://127.0.0.1:3306/bjtu_review?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true

SPRING_DATASOURCE_PASSWORD=your-db-password
JWT_SECRET=your-jwt-secret-at-least-32-chars
SPRING_MAIL_HOST=smtp.qq.com
SPRING_MAIL_PORT=587
SPRING_MAIL_USERNAME=your-email@qq.com
SPRING_MAIL_PASSWORD=your-smtp-auth-code
```

## 常见问题

### 邮件验证码发送失败

检查远端环境变量中的 SMTP 配置。QQ 邮箱必须使用 SMTP 授权码，不是 QQ 登录密码。

### 页面出现 Element Plus label 警告

这是组件库的弃用提示，不影响系统使用。后续升级 Element Plus 3 时需要将部分 `label` 作为值的写法改为 `value`。

### 接口返回 500

优先查看后端日志：

```bash
sudo journalctl -u bjtu-review --since "10 minutes ago" -p err --no-pager
```

如果是数据库结构问题，执行：

```bash
mysql -u bjtu_review -p bjtu_review < /var/www/bjtu-review/backend/src/main/resources/db/migration_align_current_schema.sql
sudo systemctl restart bjtu-review
```
