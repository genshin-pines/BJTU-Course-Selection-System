# 项目阶段性变更总结

生成时间：2026-06-11

## 阶段目标

本阶段围绕评价系统的核心可用性推进，重点解决登录状态不稳定、同账号多处登录、举报/审核治理链路不完整、点赞重复、后台审计缺失以及本地启动容易端口冲突等问题。

## 后端变更

- 登录态增加 `sessionId` 校验，学生和管理员表新增 `current_session_id`，实现同一账号后登录踢掉旧 token。
- 新增 `/api/auth/session`，供前端刷新页面后向后端确认当前 token 是否仍然有效。
- `JwtAuthenticationFilter` 增加数据库会话校验，并统一返回 JSON 401，避免认证失败误变成 403。
- 评价状态枚举和审核流转收口，评价提交后进入待审核，审核通过后发布并重新计算课程、教师评分。
- 后台审核、举报采纳、举报驳回支持填写操作原因，并写入审计日志。
- 新增 `AuditLog`、`ReviewVote` 等实体和 Mapper，补齐审计日志与点赞关系表能力。
- 点赞从简单计数调整为 `review_vote` 关系表约束，避免同一学生重复点赞。
- 新增举报和评价服务测试，覆盖主要业务分支。

## 前端变更

- 登录状态统一由 `auth` store 管理，刷新页面后通过 `/api/auth/session` 主动校验后端会话。
- 登录态存储从 `localStorage` 调整为 `sessionStorage`，降低学生端和后台在同一浏览器中互相覆盖身份的风险。
- 路由守卫按学生端和后台分别校验权限，401 跳对应登录页，403 只提示无权限。
- 写评价、点赞、举报前统一确认当前是有效学生登录态；管理员账号操作学生功能时提示切换学生账号。
- 后台审核和举报处理 UI 增加操作原因输入。
- 后台新增审计日志展示。
- 修复一批乱码中文提示和按钮文案。

## 数据库脚本

- `schema.sql` 同步增加会话字段、审计日志表、点赞关系表等结构。
- 新增 `migration_session_id.sql`，用于已有数据库补充当前会话字段。
- 新增 `migration_review_vote.sql`，用于已有数据库补充点赞关系表。
- `import_data.sql` 同步适配当前结构和状态枚举。

## 启动脚本

- 重写 `start.ps1`，启动前只清理占用项目端口 `8889`、`8081` 的进程，不再误杀所有 Java 进程。
- `start.bat` 统一调用 `start.ps1`。
- 启动日志输出到 `repo/logs/`。
- 支持 `-NoRestart`、`-SkipFrontend`、`-SkipBackend` 等参数。

## 验证情况

- 后端执行 `mvn clean test` 通过。
- 前端执行 `npm run build` 通过。
- 单处登录验证通过：同一学生账号二次登录后，旧 token 调用 `/api/auth/session` 返回 401，新 token 返回当前学生信息。
- 一键启动脚本验证通过：后端 `8889`、前端 `8081` 均可正常监听，前端首页返回 200。

## 后续注意

- 当前单处登录采用数据库 `current_session_id` 方案，尚未引入 Redis 在线会话管理。
- `sessionStorage` 能保证刷新不退登，但关闭浏览器后不会长期保留登录态。
- 现有迁移脚本需要手工执行，后续可考虑引入 Flyway 或 Liquibase。
- 项目中仍有部分历史文件存在中文编码显示异常，建议后续单独修复。
