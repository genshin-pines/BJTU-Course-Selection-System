# 需求追溯文档

## 1. 文档说明

本文档将系统功能与初始需求（IR）和用户故事（US）进行映射，实现需求到代码的双向追溯。

> **说明**：由于本项目采用 Vibe Coding 方式开发，正式的需求文档（IR/US）在项目初期未单独编写。本文档基于 README.md、使用手册及实际代码实现反推需求来源，并建立追溯矩阵。

## 4. 用户故事（US）定义

| US ID | 用户故事 | 对应 IR | 验收标准 |
|-------|----------|---------|----------|
| US-001 | 作为学生，我希望注册账号，以便使用评价系统 | IR-001, IR-003 | 填写学号、姓名、邮箱、密码后可注册；邮箱收到验证码 |
| US-002 | 作为学生，我希望登录系统，以便查看和发布评价 | IR-002, IR-004 | 学号+密码登录；新登录使旧会话失效 |
| US-003 | 作为学生，我希望搜索和筛选课程，以便找到感兴趣的课程 | IR-005, IR-006, IR-007 | 支持关键词、学院、教师、评分、标签筛选；支持多种排序 |
| US-004 | 作为学生，我希望查看课程详情和评分，以便了解课程质量 | IR-008 | 展示课程信息、教师信息、评分汇总、评价列表 |
| US-005 | 作为学生，我希望匿名评价课程，以便分享真实体验 | IR-009, IR-010, IR-012 | 评分、内容、标签、建议、考核方式；匿名发布；每课一评 |
| US-006 | 作为学生，我希望编辑或删除自己的评价，以便修正错误 | IR-011 | 仅可操作自己的评价；编辑后需重新审核 |
| US-007 | 作为学生，我希望对评价点赞或点"没用"，以便表达观点 | IR-013, IR-014 | 点击切换状态；不能给自己的评价投票 |
| US-008 | 作为学生，我希望举报违规评价，以便维护平台质量 | IR-015 | 填写举报原因；不能举报自己的评价 |
| US-009 | 作为学生，我希望申请添加未收录的课程，以便完善课程库 | IR-016 | 填写课程信息；查看申请状态 |
| US-010 | 作为学生，我希望查看自己发布的评价，以便管理 | IR-017 | 个人中心展示所有评价；支持编辑和删除 |
| US-011 | 作为管理员，我希望登录后台，以便管理平台内容 | IR-018 | 管理员专用登录入口；角色权限控制 |
| US-012 | 作为审核员，我希望审核评价，以便控制内容质量 | IR-019, IR-027 | 查看待审核列表；通过/拒绝/隐藏；填写原因 |
| US-013 | 作为审核员，我希望处理举报，以便维护平台秩序 | IR-020 | 查看待处理举报；采纳/驳回；填写处理原因 |
| US-014 | 作为维护员，我希望维护课程/教师/开课实例，以便保证数据准确 | IR-021, IR-022 | CRUD 操作；CSV 批量导入 |
| US-015 | 作为维护员，我希望管理评价标签，以便优化分类 | IR-023 | 创建和删除标签 |
| US-016 | 作为超级管理员，我希望管理管理员账号，以便控制后台权限 | IR-024, IR-025 | 创建/修改角色/重置密码/删除；三级权限 |
| US-017 | 作为管理员，我希望查看审计日志，以便追溯操作记录 | IR-026 | 展示操作时间、管理员、操作类型、原因 |
