# 项目分工文档

## 1. 团队概况

| 项目 | 内容 |
|------|------|
| 项目名称 | 北京交通大学课程评价系统（BJTU Course Review System） |
| 开发方式 | 协作开发 + Git 分支管理 |
| 团队成员 | 7 人 |

## 2. 成员分工

### 2.1 msxlz — 后端核心架构与安全

**负责模块**：后端整体框架、认证安全、管理员 RBAC、评价投票、审计日志、数据库设计

**具体工作**：
1. 后端项目框架搭建（Spring Boot + Spring Security + JWT）
2. 学生/管理员认证与会话管理（JwtAuthenticationFilter、AuthController）
3. 管理员 RBAC 三级权限模型设计与实现（SUPER_ADMIN / DEPT_OP / AUDITOR）
4. 管理员账号管理（创建/修改角色/重置密码/删除）
5. 评价投票功能（点赞/点"没用"）及匿名身份映射模型（voter_record）
6. 审计日志系统设计与实现
7. 数据库 schema 设计与迁移脚本编写
8. 敏感词过滤器核心实现
9. 全局异常处理与统一响应封装
10. 部署脚本（Windows 本地启动 + Ubuntu 远端部署）
11. 项目文档编写（README、使用手册）

**涉及的核心文件**：
- `SecurityConfig.java`、`JwtAuthenticationFilter.java`、`JwtUtils.java`
- `AdminController.java`、`AuthController.java`、`AdminServiceImpl.java`
- `Admin.java`、`AuditLog.java`、`ReviewVote.java`、`AdminMapper.java`、`AuditLogMapper.java`
- `schema.sql`、`migration_align_current_schema.sql`、`migration_admin_role.sql`、`migration_review_vote.sql`、`migration_session_id.sql`
- `start.ps1`、`start.bat`、`DEPLOY.md`、`setup-ubuntu.sh`

### 2.2 genshin-pines — 前端全栈与课程申请

**负责模块**：前端全部页面与组件、课程申请功能、学生注册、初始数据导入

**具体工作**：
1. 前端项目框架搭建（Vue 3 + Vite + Pinia + Element Plus）
2. 全部 8 个页面视图开发（Home、Login、CourseDetail、PostReview、Profile、AdminLogin、AdminDashboard、CourseApplication）
3. 全部 14 个前端子组件开发（CourseCard、CourseFilter、ReviewCard、ReviewForm、VoteButtons、RatingSummary、AppHeader、AppFooter、AdminSidebar 等）
4. 前端路由配置、Pinia 状态管理、Axios 请求封装
5. 前端 API 层封装（6 个模块：admin、course、review、report、tag、courseApplication）
6. 课程申请功能完整实现（前后端：CourseApplicationController、Service、Mapper、前端页面）
7. 学生注册功能（邮箱验证码发送与验证）
8. 系统初始版本搭建与初始数据导入
9. 评分体系设计与重构

**涉及的核心文件**：
- 全部 `frontend/src/views/` 下 8 个页面
- 全部 `frontend/src/components/` 下 14 个子组件
- `frontend/src/router/index.js`、`stores/auth.js`、`utils/request.js`
- `frontend/src/api/` 下 6 个 API 模块
- `CourseApplicationController.java`、`CourseApplicationService.java`、`CourseApplicationServiceImpl.java`
- `VerifyCodeService.java`、`RegisterRequest.java`、`SendCodeRequest.java`
- `migration-course-application.sql`
