## 2. 冒烟测试

### 2.1 测试脚本

项目提供冒烟测试脚本 `scripts/smoke-test.ps1`（通过 `smoke-test.bat` 调用），覆盖以下 API 接口：

| 序号 | 测试项 | 接口路径 | 类型 | 状态 |
|------|--------|----------|------|------|
| 1 | 课程搜索 | GET /course/search?page=1&size=5 | 公共 | ✅ |
| 2 | 课程详情 | GET /course/instances/{id}/detail | 公共 | ✅ |
| 3 | 评价列表 | GET /review/instance/{id} | 公共 | ✅ |
| 4 | 学生登录 | POST /auth/login | 学生 | ✅ |
| 5 | 学生会话验证 | GET /auth/session | 学生 | ✅ |
| 6 | 学生投票状态 | GET /review/instance/{id}/liked | 学生 | ✅ |
| 7 | 学生点"没用"状态 | GET /review/instance/{id}/downvoted | 学生 | ✅ |
| 8 | 点赞切换（可选） | POST /review/like/{id} | 学生 | ✅ |
| 9 | 管理员登录 | POST /auth/admin/login | 管理员 | ✅ |
| 10 | 管理员会话验证 | GET /auth/session | 管理员 | ✅ |
| 11 | 待审核评价列表 | GET /admin/reviews/pending | 管理员 | ✅ |
| 12 | 举报列表 | GET /admin/reports | 管理员 | ✅ |
| 13 | 审计日志 | GET /admin/audit-logs | 管理员 | ✅ |

### 2.2 使用方法

```powershell
# 基础测试（只读接口）
.\smoke-test.bat

# 包含写入操作测试（点赞切换）
.\smoke-test.bat -IncludeWriteChecks

# 指定后端地址
.\smoke-test.bat -BaseUrl http://localhost:8889/api
```

## 6. 测试结论

### 6.1 测试结论

- **冒烟测试**：13/13 全部通过，核心 API 接口正常工作
- **手动功能测试**：63/63 全部通过，所有功能模块正常工作
- **权限控制**：5/5 全部通过，三级权限隔离有效

### 6.2 建议

1. **补充单元测试**：当前项目缺少后端单元测试，建议补充 Service 层和 Controller 层的单元测试。
2. **集成测试**：建议增加前后端联调的集成测试，覆盖完整的业务流程。
3. **性能测试**：建议对课程搜索接口进行性能测试，验证大数据量下的响应时间。
4. **安全测试**：建议进行安全渗透测试，检查 XSS、CSRF、SQL 注入等常见漏洞。
5. **自动化回归测试**：建议将冒烟测试集成到 CI/CD 流程中，每次部署前自动运行。
